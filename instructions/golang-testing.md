# Golang Testing - Skill Instructions

## Focused Area

Complete Go testing patterns following TDD methodology.

## Content Strategy

This skill covers practical testing approaches:

1. **TDD**: RED-GREEN-REFACTOR cycle
2. **Table-Driven Tests**: Standard Go pattern
3. **Subtests**: Organizing and parallel execution
4. **Mocking**: Interface-based mocking
5. **Benchmarks**: Performance testing
6. **Fuzzing**: Input validation testing
7. **Coverage**: Code coverage analysis

## Language Choice

**Chinese (zh)** - Accessible testing guidance.

## Update Guidelines

1. Include practical, copy-pasteable examples
2. Cover both basic and advanced testing
3. Emphasize Go idioms over other languages
4. Include coverage targets and best practices
